# R2

All media is stored in existing private Cloudflare R2 buckets.
The database stores **object keys only** (no permanent public URLs).
Signed URLs are temporary and generated by backend services.

## Buckets (existing)
- `mrq-music-masters`
- `mrq-music-previews`
- `mrq-music-normalized`
- `mrq-music-artwork`
- `mrq-music-exports`

## Staging model (no separate bucket)
`staging` is a logical upload type, not a physical bucket.

- Logical type: `staging`
- Physical bucket: `mrq-music-masters`
- Key format: `staging/uploads/{trackId}/{safeFileName}`

## Key patterns
- Staging source audio:
  - `staging/uploads/{trackId}/{safeFileName}`
- Canonical original in masters:
  - `masters/{artistId}/{trackId}/original/{safeFileName}`
- Normalized outputs:
  - `streaming/{trackId}/master.flac`
  - `radio/{trackId}/radio.flac`
- Preview:
  - `previews/{trackId}/preview.mp3`
- Waveform (optional):
  - `waveforms/{trackId}/waveform.json`
- Artwork:
  - `tracks/{trackId}/{safeFileName}`
  - `artists/{artistId}/{safeFileName}`
  - `albums/{albumId}/{safeFileName}`
- Exports:
  - `releases/{releaseId}/{safeFileName}`
